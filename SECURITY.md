# Security Policy

## Supported Versions

Security fixes are released for the latest PortKill release.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub Security Advisories when available:

https://github.com/mr-tanta/portkill/security/advisories/new

If that flow is unavailable, open a private report with enough detail to reproduce the issue, including:

- PortKill version
- Operating system
- Command and flags used
- Expected behavior
- Observed behavior
- Any relevant logs or proof of concept

Do not post public exploit details until a fix is available.

## Security Checks

The repository runs automated checks for shell quality, secret exposure, filesystem vulnerabilities, release packaging, and workflow security posture on pushes, pull requests, weekly schedules, and manual dispatches.
